# BOOTSTRAP.md - Work OpenClaw Setup Guide

**Delete this file after completing setup.**

## Setup Checklist

### 1. Customize Core Files
- [ ] **USER.md** - Add your actual work context, stakeholders, projects
- [ ] **IDENTITY.md** - Choose your work assistant name and persona  
- [ ] **MEMORY.md** - Replace templates with real work information
- [ ] **TOOLS.md** - Add your actual work infrastructure details
- [ ] **HEARTBEAT.md** - Set your business hours and monitoring preferences

### 2. Set Up Authentication
- [ ] Configure work email access (if applicable)
- [ ] Set up calendar integration (if applicable) 
- [ ] Configure any work API keys (store securely)
- [ ] Test access to work systems through OpenClaw

### 3. Configure Skills
- [ ] Review available skills for work relevance
- [ ] Install/configure work-specific skills as needed
- [ ] Test key workflows (email, calendar, document access)

### 4. Set Up Backup System
- [ ] Copy backup scripts from personal setup
- [ ] Adapt backup locations for work environment
- [ ] Test backup and restore procedures
- [ ] Schedule regular backups (if appropriate)

### 5. Security Review
- [ ] Ensure no personal credentials/data in work setup
- [ ] Verify work data doesn't leak to personal systems
- [ ] Check that backup locations are work-appropriate
- [ ] Review sharing and external access permissions

### 6. Test Core Workflows
- [ ] Session startup sequence (reads correct files)
- [ ] Heartbeat monitoring (appropriate for work hours)
- [ ] Rate limit monitoring (works with work model setup)
- [ ] Memory updates (captures work context appropriately)

## Initial Memory Population

### Day 1 Tasks
1. Create `memory/[YYYY-MM-DD].md` for today
2. Document initial work context and current projects  
3. Add key stakeholder information to MEMORY.md
4. Test and refine communication preferences

### Week 1 Tasks
1. Refine heartbeat monitoring based on actual work patterns
2. Build out project-specific context in memory files
3. Adjust SOUL.md based on work culture fit
4. Document any lessons learned about work integration

## Ongoing Maintenance

### Weekly
- Review and update project status in MEMORY.md
- Clean up completed tasks from memory files
- Backup work context (if appropriate for your environment)

### Monthly  
- Review and refine work communication patterns
- Update stakeholder information as relationships evolve
- Assess and adjust monitoring/heartbeat patterns

---

**Remember:** This template is based on proven patterns from a personal OpenClaw setup. Adapt everything for your specific work environment, company culture, and professional needs.
